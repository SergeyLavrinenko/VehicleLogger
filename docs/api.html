VehicleLogger API

Документация REST API бэкенда. Base URL: https://api.vehiclelogger.ru

Аутентификация:
Устройства: заголовок X-Device-Key: <api-key>
Пользователи (веб-панель): заголовок Authorization: Bearer <jwt-token>
Провизионинг: серийный номер + секрет устройства в теле запроса

Устройство → Бэкенд

Эндпоинты, вызываемые ESP32. Аутентификация через X-Device-Key.

POST /api/telemetry X-Device-Key Отправка телеметрии CAN-шины ▶
Заголовки
Content-Typeapplication/json
X-Device-Key<api-key>
Тело запроса
{
  "deviceId": "VL-A3F82B01",
  "timestamp": "2026-03-29T14:30:00Z",
  "data": {
    "rpm": 1800,
    "speed": 85,
    "coolantTemp": 92,
    "oilPressure": 3.5,
    "fuelLevel": 45,
    "voltage": 13.8,
    "dtcCodes": ["P0301"]
  }
}
Параметры
deviceId string required Серийный номер устройства
timestamp string (ISO 8601) required Время замера (UTC)
data.rpm number Обороты двигателя
data.speed number Скорость (км/ч)
data.coolantTemp number Температура ОЖ (°C)
data.oilPressure number Давление масла (бар)
data.fuelLevel number Уровень топлива (%)
data.voltage number Напряжение бортсети (В)
data.dtcCodes string[] Коды ошибок OBD-II
Ответы
КодОписание
200Телеметрия сохранена
401Неверный или отсутствующий API-ключ
400Некорректный формат JSON
POST /api/device/ping X-Device-Key Heartbeat — устройство онлайн ▶
Заголовки
X-Device-Key<api-key>
Тело запроса
{
  "deviceId": "VL-A3F82B01"
}
Ответы
КодОписание
200OK. Обновлены IsOnline = true, LastPingAt = now
401Неверный API-ключ
Устройство отправляет heartbeat параллельно с телеметрией. Если heartbeat не приходит >60 сек, устройство считается оффлайн.

Провизионинг

Регистрация и привязка устройств к аккаунтам. См. firmware/PROVISIONING.md.

POST /api/devices/provision Device Secret Получение API-ключа (вызывает ESP32) ▶
Описание

ESP32 вызывает этот эндпоинт после настройки WiFi для получения API-ключа. Устройство поллит каждые 10 секунд, пока владелец не привяжет его через claim.

Тело запроса
{
  "serialNumber": "VL-A3F82B01",
  "deviceSecret": "a1b2c3d4e5f6...полный_секрет_64_hex"
}
Ответы
КодОписаниеТело ответа
200 Устройство привязано, ключ готов {"apiKey":"...","backendUrl":"https://...","sendIntervalMs":5000}
202 Ещё не привязано, ждём claim {"status":"waiting_for_claim"}
403 Неверный секрет устройства {"error":"invalid_secret"}
410 Устройство деактивировано {"error":"deactivated"}
Только HTTPS. Полный секрет (256 бит) подтверждает физическое владение устройством.
POST /api/devices/claim JWT Bearer Привязка устройства к аккаунту ▶
Описание

Вызывается из веб-панели после сканирования QR-кода на устройстве. Требует JWT авторизацию (роль admin или fleet_manager).

Тело запроса
{
  "serialNumber": "VL-A3F82B01",
  "deviceSecretFragment": "A1B2C3D4",
  "vehicleId": "uuid-или-null"
}
Параметры
serialNumber string required Серийный номер с QR-кода
deviceSecretFragment string required Первые 8 hex-символов секрета (из QR)
vehicleId uuid | null ID фуры для привязки (можно назначить позже)
Ответы
КодОписание
200Успех: {"deviceId":"uuid","serialNumber":"VL-A3F82B01","status":"claimed"}
400Фрагмент секрета не совпадает
404Серийный номер не найден
409Устройство уже привязано к другому тенанту
Бэкенд: генерирует API-ключ, сохраняет хеш, создаёт provisioning_token (TTL 10 мин), обновляет tenant_id и status.
POST /api/devices/{id}/unclaim JWT Bearer Отвязка устройства ▶
Описание

Отвязывает устройство от аккаунта. Используется при продаже/передаче устройства. Очищает tenant_id, vehicle_id, api_key_hash. Статус → manufactured.

Ответы
КодОписание
200Устройство отвязано
403Устройство принадлежит другому тенанту
404Устройство не найдено
PUT /api/devices/{id}/vehicle JWT Bearer Назначить устройство на фуру ▶
Тело запроса
{
  "vehicleId": "uuid-фуры"
}
Ответы
КодОписание
200Устройство назначено на фуру
403Устройство или фура принадлежат другому тенанту
404Устройство или фура не найдены
POST /api/devices/{id}/rotate-key JWT Bearer Ротация API-ключа ▶
Описание

Генерирует новый API-ключ. Старый ключ перестаёт работать. Устройство получит 401 → сбросит ключ → перейдёт в режим поллинга → получит новый ключ через /provision.

Ответы
КодОписание
200Новый ключ сгенерирован. Устройство обновится автоматически.
403Устройство принадлежит другому тенанту

Авторизация

JWT-аутентификация для веб-панели.

POST /api/auth/login Public Вход в систему, получение JWT ▶
Тело запроса
{
  "email": "admin@example.com",
  "password": "********"
}
Успешный ответ (200)
{
  "token": "eyJhbGciOiJIUzI1NiIs...",
  "expiresAt": "2026-03-30T14:30:00Z",
  "user": {
    "id": "uuid",
    "role": "admin",
    "tenantId": "uuid"
  }
}
JWT содержит: userId, role, tenantId. Middleware фильтрует все данные по tenantId.

Telegram-бот

Webhook для команд водителя.

POST /api/telegram/webhook Telegram Bot Token Webhook для Telegram Update ▶
Команды
КомандаДействие
/startПривязка Telegram ID к водителю
/refuel <литры> <руб>Сохранение заправки
/statusТекущие параметры фуры водителя
/alertsПоследние алерты по фуре
Водитель идентифицируется по Telegram ID, привязанному к записи User. Видит данные только своей фуры.
