# Устройство отслеживания состояния автомобиля (Автологгер)

## Описание

Автологгер — это телематическая система для мониторинга состояния автомобиля в реальном времени. Устройство подключается к CAN-шине (через стандартный OBD-II разъем) и автоматически собирает данные о параметрах работы двигателя, пробеге, ошибках и других показателях.  
Система состоит из трех компонентов:  
- Аппаратное устройство на базе ESP32-C3 + CAN-трансивер.  
- Облачный сервер для приема, хранения и обработки телеметрии.  
- Клиентские интерфейсы: веб-панель и Telegram-бот для доступа к данным.  

Устройство работает автономно после установки, не требует технических знаний от пользователя и передает данные на сервер через Wi‑Fi (в базовой версии) или LTE-модем (в расширенной версии).

## Функции

### Мониторинг параметров автомобиля

Система в реальном времени отслеживает основные параметры через OBD-II протокол:  
- Скорость движения  
- Обороты двигателя  
- Температура охлаждающей жидкости  
- Уровень топлива и расчетный расход  
- Одометр (общий пробег)  
- Напряжение бортовой сети  
- Статус систем автомобиля  

Данные собираются с частотой 1–5 секунд и буферизуются на устройстве для надежной отправки на сервер даже при временных обрывах связи.

### Диагностика и ошибки

Устройство автоматически считывает диагностические коды неисправностей (DTC) и передает их на сервер с расшифровкой:  
- Обнаружение новых ошибок Check Engine  
- Хранение истории ошибок с привязкой к времени и пробегу  
- Человекочитаемое описание кодов ошибок  
- Приоритизация: критические / важные / информационные ошибки  

При появлении новой ошибки пользователь получает push‑уведомление через Telegram или веб‑интерфейс.

### Журнал поездок

Сервер автоматически группирует телеметрию в логические поездки на основе событий включения/выключения зажигания и длительных остановок:  
- Время начала и окончания поездки  
- Пройденное расстояние  
- Средняя и максимальная скорость  
- Расход топлива за поездку  
- Статистика по агрессивной езде (резкие ускорения/торможения)  

Веб-панель показывает список поездок с возможностью фильтрации по дате и экспорта отчетов.

### Геолокация (опционально)

При наличии GPS-модуля система обеспечивает:  
- Отображение текущего местоположения автомобиля на карте  
- Трек поездок с визуализацией маршрута  
- Настройка геозон с уведомлениями о въезде/выезде  
- Определение адреса остановок (парковка дома/работа)  

### Уведомления и алерты

Система поддерживает настраиваемые уведомления через Telegram-бот:  
- Новые ошибки двигателя  
- Начало движения автомобиля  
- Превышение заданной скорости  
- Выезд из разрешенной геозоны  
- Напоминания о плановом ТО по пробегу  
- Низкий заряд аккумулятора устройства  
- Запись данных о заправке  

### Аналитика и отчеты

Веб-панель предоставляет аналитические инструменты:  
- Графики изменения параметров во времени (температура, обороты, скорость)  
- Статистика расхода топлива по дням/неделям/месяцам  
- Анализ стиля вождения  
- Сводные отчеты по пробегу и времени использования  
- Экспорт данных в CSV/Excel для внешнего анализа

## Применение

### Личное использование

#### Контроль состояния автомобиля

Владелец личного автомобиля получает полную картину эксплуатации транспорта:  
- Постоянный мониторинг технического состояния  
- Раннее обнаружение неисправностей до визита в сервис  
- История обслуживания и ошибок для диагностики хронических проблем  
- Понимание реального расхода топлива и факторов, влияющих на него  

#### Экономия на обслуживании

Система помогает оптимизировать расходы на эксплуатацию:  
- Точный учет пробега для планирования ТО по регламенту  
- Снижение расхода топлива через анализ стиля вождения  

#### Безопасность и контроль

При передаче автомобиля третьим лицам (родственники, друзья, аренда):  
- Отслеживание местоположения в реальном времени  
- Уведомления о начале движения  
- Контроль соблюдения договоренностей (геозоны, время использования)  
- История поездок для разрешения спорных ситуаций  

### Малый автопарк (5–50 машин)

#### Управление парком

Для компаний с небольшим автопарком (доставка, сервис, монтажники) система предоставляет:  
- Единый дашборд с состоянием всех автомобилей  
- Мониторинг местоположения и маршрутов сотрудников  
- Учет реального пробега и времени работы  
- Планирование ТО и ремонтов на основе фактических данных  

#### Оптимизация затрат

Детальная аналитика позволяет снизить операционные расходы:  
- Выявление нецелевого использования (личные поездки, простои с включенным двигателем)  
- Анализ расхода топлива по машинам и водителям  
- Обнаружение агрессивного стиля вождения, ведущего к износу  
- Оптимизация маршрутов на основе истории поездок  

#### Контроль водителей

Система обеспечивает прозрачность использования транспорта:  
- Объективная оценка работы водителей (время в пути, простои)  
- Мотивация через рейтинги по стилю вождения и расходу  
- Защита от злоупотреблений (ночные поездки, выезды за пределы региона)  
- Автоматические отчеты для бухгалтерии и учета  

### Прототипирование и разработка

#### Реверс-инжиниринг CAN-протоколов

Устройство на ESP32-C3 с открытой прошивкой может использоваться для изучения автомобильных протоколов:  
- Снятие и анализ сырых CAN-кадров  
- Декодирование проприетарных сообщений конкретных моделей  
- Разработка кастомных интеграций с нестандартными системами  

#### Образовательные цели

Проект подходит для изучения современных технологий IoT:  
- Работа с автомобильными протоколами (CAN, OBD-II)  
- Разработка встраиваемых систем на ESP32  
- Построение облачной архитектуры для телеметрии  
- Создание веб-интерфейсов и чат-ботов

## Стек технологий

### Аппаратная часть

#### Микроконтроллер

ESP32-C3 выбран как основа устройства благодаря оптимальному соотношению возможностей и стоимости:  
- Встроенный CAN-контроллер (TWAI) для работы с автомобильной шиной без внешних микросхем-контроллеров  
- RISC-V ядро 160 МГц с ~400 КБ SRAM — достаточно для буферизации и обработки телеметрии  
- Wi‑Fi 802.11b/g/n и Bluetooth 5.0 для беспроводной связи  
- Низкое энергопотребление и режимы глубокого сна  
- Поддержка OTA-обновлений прошивки  
- Богатая экосистема библиотек (ESP-IDF, Arduino)  

#### CAN-трансивер

Для физического подключения к CAN-шине используется внешний трансивер:  
- SN65HVD230 или TJA1050 — стандартные 3.3V/5V CAN-трансиверы  
- Преобразование логических сигналов TWAI в дифференциальные CAN_H/CAN_L  
- Защита от помех и перенапряжений на линии  

#### Питание

Устройство питается от бортовой сети автомобиля через OBD-II разъем:  
- DC-DC преобразователь 12 V → 5 V → 3.3 V  
- Защита от обратной полярности и скачков напряжения (TVS-диоды)  
- Фильтрация помех от системы зажигания  
- Опционально: резервный аккумулятор для сохранения данных при отключении питания  

#### Опциональные модули

Для расширенных версий устройства:  
- GPS/GNSS модуль (NEO-6M, NEO-M8N) для геолокации  
- LTE/2G модем (SIM800, SIM7600) для связи вне Wi‑Fi зоны  
- Акселерометр (например, MPU6050) для детекции резких маневров и аварий  
- microSD для локального резервного хранения данных  

### Серверная часть

#### Backend

- C# (например, ASP.NET Core) для реализации API, обработки телеметрии и фоновых задач.  

#### База данных

- PostgreSQL — основная БД для сущностей (users, devices, vehicles, trips, alerts).  
- InfluxDB или другая time-series БД — хранение телеметрии с автоматической агрегацией.  

### Веб-интерфейс

#### Frontend Framework

- Современный SPA-фреймворк (React, Vue или Svelte) — выбор фиксируется на этапе реализации.  

#### Основные экраны

1. Dashboard: обзор всех автомобилей, текущий статус, последние алерты.  
2. Vehicle Details: подробная информация по выбранной машине, карта, графики.  
3. Trips Log: таблица поездок с фильтрацией, детализация каждой поездки с треком.  
4. Analytics: сводные отчеты, графики потребления, сравнение периодов.  
5. Settings: управление устройствами, геозонами, настройка уведомлений.  

### Telegram-бот

- Использование Telegram Bot API для чат-бота, который предоставляет быстрый доступ к статусу автомобиля, последним поездкам и уведомлениям.
