# Сергей — Устройство (ESP32)

## Зона ответственности

Разработка прошивки и аппаратной части устройства на базе ESP32-C3, которое устанавливается на фуру и передаёт данные CAN-шины на бэкенд.

---

## Что делает

### Аппаратная часть
- Схема подключения ESP32-C3 + CAN-трансивер (SN65HVD230 / TJA1050)
- Подключение к CAN-шине через OBD-II / прямое подключение
- Питание от бортовой сети (12V → 3.3V, DC-DC)
- Опционально: подключение GPS, LTE-модема

### Прошивка (C/C++, ESP-IDF / Arduino)
- Инициализация CAN-контроллера (TWAI) и чтение кадров с шины
- Парсинг OBD-II параметров: обороты, температура, скорость, давление масла, уровень топлива, напряжение, DTC-коды
- Формирование JSON-пакета телеметрии
- Отправка данных на бэкенд по HTTP/HTTPS (`POST /api/telemetry`)
- Heartbeat (`POST /api/device/ping`) — сигнал «устройство онлайн»
- Аутентификация на бэкенде через API-ключ в заголовке `X-Device-Key`

### Буферизация и надёжность
- Локальный буфер данных при потере Wi-Fi/LTE
- Повторная отправка буферизованных данных при восстановлении связи
- Интервал отправки: каждые 1–5 секунд

### Связь
- Настройка Wi-Fi подключения (базовая версия)
- Опционально: LTE-модем для работы вне зоны Wi-Fi

## Интерфейсы (с чем взаимодействует)

| Направление       | Куда / Откуда    | Протокол | Что передаёт                      |
|--------------------|------------------|----------|-----------------------------------|
| ESP32 → CAN-шина  | Фура             | CAN/TWAI | Чтение параметров                 |
| ESP32 → Бэкенд    | .NET Backend     | HTTP     | JSON с телеметрией                |
| ESP32 → Бэкенд    | .NET Backend     | HTTP     | Heartbeat (ping)                  |

---

## Зависимости от других участников

- **Тимур (бэкенд):** Сергей отправляет данные на эндпоинты, которые реализует Тимур. Нужно согласовать формат JSON-пакета телеметрии и механизм аутентификации (`X-Device-Key`).

---

## Не входит в зону ответственности

- Бэкенд, база данных, API
- Фронтенд (веб-панель)
- Telegram-бот
- AI-анализ данных
