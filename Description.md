# VehicleLogger — Система мониторинга состояния фур

## Описание

Телематическая система для мониторинга грузовых автомобилей (фур) в реальном времени. Устройство подключается к CAN-шине фуры, считывает параметры и отправляет данные на сервер для анализа.

---

## Компоненты системы

### 1. Устройство (ESP32)

Аппаратный модуль, устанавливаемый на каждую фуру.

- Подключается к CAN-шине через OBD-II / прямое подключение
- Считывает параметры: обороты, температура, скорость, давление масла, уровень топлива, коды ошибок (DTC) и др.
- Буферизует данные при потере связи
- Отправляет телеметрию на бэкенд по Wi-Fi / LTE
- Идентифицируется уникальным ID, привязанным к конкретной фуре

### 2. Веб-интерфейс (Администратор)

Панель управления для диспетчера / владельца автопарка.

**Бэкенд (.NET):**
- Приём и хранение телеметрии от устройств
- REST API для фронтенда и Telegram-бота
- Интеграция с нейросетью (через API) для анализа данных CAN-шины
- Генерация алертов при обнаружении проблем

**Фронтенд (Vue 3):**
- Список всех подключённых фур с текущим статусом
- Графики параметров CAN-шины для каждой фуры (обороты, температура, давление, топливо и т.д.)
- Просмотр алертов и результатов AI-анализа
- Просмотр данных о заправках, внесённых водителями
- Администратор **только просматривает** данные, не вносит их

### 3. Telegram-бот (Водитель)

Интерфейс для водителя конкретной фуры.

**Ввод данных:**
- Внесение заправки: объём топлива, стоимость, АЗС
- Данные отображаются у администратора в веб-панели

**Получение алертов:**
- Уведомления о проблемах, обнаруженных на CAN-шине (перегрев, ошибки двигателя, низкое давление масла и т.д.)
- Алерты формируются нейросетью на бэкенде

**Ограничения:**
- Водитель видит и вносит данные **только по своей фуре**

## Роли и права доступа

| Роль          | Просмотр данных      | Внесение данных | Область видимости |
|---------------|----------------------|-----------------|-------------------|
| Администратор | Все фуры             | Нет             | Весь автопарк     |
| Водитель      | Только своя фура     | Да (заправки)   | Своя фура         |
| Устройство    | —                    | Телеметрия CAN  | Своя фура         |

## Поток данных

```
┌─────────────┐         ┌─────────────────┐         ┌─────────────────┐
│  ESP32      │────────▶│  .NET Backend   │────────▶│  Vue 3          │
│  (CAN-шина) │  HTTP   │  (API + AI)     │         │  (Администратор)│
└─────────────┘         └────────┬────────┘         └─────────────────┘
                                 │
                                 │  Алерты
                                 ▼
                        ┌─────────────────┐
                        │  Telegram-бот   │
                        │  (Водитель)     │──── Заправки ────▶ Бэкенд
                        └─────────────────┘
```

1. **ESP32** считывает данные с CAN-шины → отправляет на бэкенд
2. **Бэкенд** сохраняет телеметрию в PostgreSQL, прогоняет через нейросеть (API) для анализа
3. **Нейросеть** выявляет аномалии и проблемы → бэкенд формирует алерты
4. **Алерты** уходят водителю в Telegram и отображаются в веб-панели
5. **Водитель** вносит данные о заправках через Telegram-бот → данные видны администратору
6. **Администратор** просматривает всё в веб-панели: телеметрию, графики, алерты, заправки

---

## AI-анализ (Нейросеть)

Бэкенд обращается к нейросети через API для:

- **Детекция аномалий:** выявление отклонений параметров CAN-шины от нормы (перегрев, нетипичные обороты, падение давления)
- **Диагностика проблем:** интерпретация DTC-кодов и комбинаций параметров в человекочитаемые описания проблем
- **Приоритизация алертов:** определение критичности обнаруженных проблем (критический / предупреждение / информационный)
- **Рекомендации:** предложения по действиям (остановить движение, проверить при ТО, наблюдать)

Результаты анализа:
- Отображаются в веб-панели администратора
- Отправляются водителю в Telegram (критические и предупреждения)

## Сущности данных

### Фура (Vehicle)
- ID, название/номер, привязанное устройство, привязанный водитель

### Устройство (Device)
- ID, серийный номер, привязанная фура, статус (онлайн/оффлайн)

### Телеметрия (Telemetry)
- ID устройства, timestamp, параметры CAN-шины (обороты, температура, скорость, давление масла, уровень топлива, напряжение, DTC-коды)

### Заправка (Refueling)
- ID фуры, водитель, timestamp, объём (л), стоимость (₽), АЗС (опционально)

### Алерт (Alert)
- ID фуры, timestamp, уровень критичности, описание проблемы, источник (AI-анализ), статус (новый/просмотрен)

### Пользователь (User)
- ID, роль (admin / driver), привязанная фура (для водителя), Telegram ID (для водителя)
