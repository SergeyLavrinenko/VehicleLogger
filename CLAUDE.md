## Проект

VehicleLogger — система мониторинга состояния фур в реальном времени. Подробное описание в `Description.md`.

**Статус:** Этап планирования, реализация не начата.

## Архитектура (три компонента)

1. **Устройство (ESP32-C3)** — C/C++ (ESP-IDF/Arduino). Считывает данные с CAN-шины фуры, буферизует и отправляет на бэкенд по Wi-Fi/LTE.
2. **Веб-панель (Администратор)** — .NET бэкенд (ASP.NET Core) + Vue 3 фронтенд. REST API, хранение телеметрии в PostgreSQL, AI-анализ данных через API нейросети, отображение графиков и алертов.
3. **Telegram-бот (Водитель)** — ввод заправок (объём, стоимость), получение алертов о проблемах. Водитель видит только свою фуру.

## Роли

- **Администратор** — просматривает данные всех фур, не может вносить данные
- **Водитель** — вносит заправки и просматривает данные только по своей фуре
- **Устройство** — отправляет телеметрию CAN-шины на бэкенд

## Поток данных

```
ESP32 (CAN) → .NET API → PostgreSQL → Vue 3 (админ)
                  ↓
             AI-анализ → Алерты → Telegram (водитель) → Заправки → Бэкенд
```
