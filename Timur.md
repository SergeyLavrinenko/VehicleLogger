# Тимур — Веб-интерфейс (Бэкенд + Фронтенд)

## Зона ответственности

Разработка серверной части (.NET) и веб-панели администратора (Vue 3). Бэкенд является центральным узлом системы — принимает телеметрию, хранит данные, интегрируется с AI и обслуживает все клиенты.

---

## Что делает

### Бэкенд (.NET, ASP.NET Core, C#)

**Приём телеметрии от устройств:**
- `POST /api/telemetry` — приём JSON-пакетов от ESP32
- `POST /api/device/ping` — heartbeat, обновление статуса устройства (онлайн/оффлайн)
- Аутентификация устройств по API-ключу (`X-Device-Key`)
- Сохранение телеметрии в PostgreSQL

**REST API для фронтенда:**
- `GET /api/vehicles` — список всех фур с текущим статусом
- `GET /api/vehicles/{id}` — детали по фуре
- `GET /api/vehicles/{id}/telemetry` — телеметрия с фильтрацией по времени
- `GET /api/vehicles/{id}/alerts` — алерты по фуре
- `GET /api/vehicles/{id}/refuels` — заправки по фуре
- `GET /api/alerts` — все алерты по автопарку
- `GET /api/dashboard` — сводка для дашборда
- `POST /api/auth/login` — авторизация администратора (JWT)

**REST API для Telegram-бота:**
- `POST /api/telegram/webhook` — приём команд от водителей через Telegram
- Обработка команд: `/start`, `/refuel`, `/status`, `/alerts`
- Отправка алертов и подтверждений водителям через Telegram Bot API (`sendMessage`)

**WebSocket:**
- `wss://server/ws/dashboard` — пуш обновлений на фронтенд в реальном времени
- События: `telemetry.update`, `alert.new`, `device.status`, `refuel.new`

**Интеграция с AI API:**
- Батчевая отправка телеметрии на анализ в нейросеть (Claude / OpenAI)
- Обработка ответа: создание алертов, отправка в Telegram, пуш на фронтенд
- Приоритизация: критический / предупреждение / информационный

**База данных (PostgreSQL):**
- Проектирование схемы: таблицы `vehicles`, `devices`, `telemetry`, `refuels`, `alerts`, `users`
- Миграции
- Запросы для агрегации и аналитики

### Фронтенд (Vue 3 + Vite)

**Экраны:**
- **Dashboard** — обзор всех фур, текущий статус, последние алерты
- **Детали фуры** — графики параметров CAN-шины (обороты, температура, давление, топливо)
- **Алерты** — список проблем с AI-анализом и рекомендациями
- **Заправки** — данные, внесённые водителями через Telegram

**Функциональность:**
- Авторизация администратора (JWT)
- Подключение к WebSocket для обновлений в реальном времени
- Графики и визуализация телеметрии
- Фильтрация данных по времени и фуре
- Администратор только просматривает данные, не вносит

## Интерфейсы (с чем взаимодействует)

| Направление          | Куда / Откуда     | Протокол        | Что передаёт                      |
|-----------------------|-------------------|-----------------|-----------------------------------|
| ESP32 → Бэкенд       | От Сергея         | HTTP             | JSON телеметрия, heartbeat       |
| Telegram → Бэкенд    | От Пайрава        | HTTPS (webhook)  | Команды водителей                |
| Бэкенд → Telegram    | К Пайраву         | HTTPS            | Алерты, подтверждения            |
| Бэкенд → AI API      | Claude / OpenAI   | HTTPS            | Батч телеметрии → аномалии      |
| Бэкенд → PostgreSQL  | БД                | TCP              | SQL запросы                      |
| Бэкенд → Фронтенд    | Свой фронтенд     | WebSocket        | Обновления в реальном времени    |
| Фронтенд → Бэкенд    | Свой бэкенд       | HTTPS (REST)     | Запросы данных                   |

---

## Зависимости от других участников

- **Сергей (устройство):** Тимур принимает телеметрию от ESP32. Нужно согласовать формат JSON и API-ключи.
- **Пайрав (Telegram-бот):** Тимур предоставляет webhook-эндпоинт и API для бота. Нужно согласовать формат команд и ответов.
