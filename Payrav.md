# Пайрав — Telegram-бот (Водитель)

## Зона ответственности

Разработка Telegram-бота для водителей. Бот позволяет водителю вносить данные о заправках, просматривать статус своей фуры и получать алерты о проблемах.

---

## Что делает

### Команды бота

| Команда                 | Описание                                           |
|-------------------------|-----------------------------------------------------|
| `/start`                | Регистрация — привязка Telegram ID к учётной записи водителя |
| `/refuel <литры> <руб>` | Внесение заправки (объём, стоимость, опционально АЗС) |
| `/status`               | Текущие параметры фуры (обороты, температура, скорость, топливо) |
| `/alerts`               | Последние алерты по своей фуре                     |

### Входящие уведомления (от бэкенда)

Водитель получает push-сообщения при:
- Обнаружении проблемы на CAN-шине (перегрев, ошибки двигателя, низкое давление масла)
- Критических и предупреждающих алертах от AI-анализа
- Подтверждении сохранённой заправки

### Логика и ограничения

- Водитель видит и вносит данные **только по своей фуре**
- Идентификация водителя по Telegram ID, привязанному к записи User в БД
- Бот не даёт доступ к данным других фур
- Бот не предоставляет функций администратора

### Взаимодействие с бэкендом

Бот работает через webhook — Telegram отправляет обновления на бэкенд, бэкенд обрабатывает и отвечает:

```
Водитель → Telegram → Webhook (POST /api/telegram/webhook) → .NET Backend → PostgreSQL
                                                                    ↓
                                                              Ответ водителю
                                                           (Telegram sendMessage)
```

**Входящие (Telegram → Бэкенд):**
- Все команды водителя приходят на `POST /api/telegram/webhook`

**Исходящие (Бэкенд → Telegram):**
- Алерты и ответы отправляются через Telegram Bot API метод `sendMessage`

---

## Интерфейсы (с чем взаимодействует)

| Направление           | Куда / Откуда     | Протокол          | Что передаёт                      |
|------------------------|-------------------|--------------------|-----------------------------------|
| Водитель → Telegram    | Telegram          | Telegram UI        | Команды: /refuel, /status и др.  |
| Telegram → Бэкенд     | К Тимуру          | HTTPS (webhook)    | JSON с командой водителя         |
| Бэкенд → Telegram     | От Тимура         | HTTPS              | Алерты, ответы на команды        |
| Telegram → Водитель    | Telegram          | Telegram UI        | Сообщения бота                   |

---

## Зависимости от других участников

- **Тимур (бэкенд):** Пайрав зависит от webhook-эндпоинта и API, которые реализует Тимур. Нужно согласовать:
  - Формат данных, приходящих на webhook
  - Формат ответов на команды `/status` и `/alerts`
  - Формат отправки заправки (`/refuel`)
  - Механизм привязки Telegram ID к водителю

---

## Не входит в зону ответственности

- Прошивка ESP32, аппаратная часть
- Бэкенд, база данных, REST API
- Фронтенд (веб-панель администратора)
- AI-анализ данных
