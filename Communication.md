## 1. ESP32 → Бэкенд

**Протокол:** HTTP/HTTPS
**Формат:** JSON
**Направление:** Устройство → Бэкенд (однонаправленное)
**Инициатор:** ESP32

### Что передаётся

| Эндпоинт              | Метод | Описание                          |
|------------------------|-------|-----------------------------------|
| `POST /api/telemetry`  | POST  | Пакет телеметрии с данными CAN    |
| `POST /api/device/ping`| POST  | Heartbeat — устройство онлайн     |

### Формат пакета телеметрии

```json
{
  "deviceId": "ESP32-001",
  "timestamp": "2026-03-22T14:30:00Z",
  "data": {
    "rpm": 1800,
    "speed": 85,
    "coolantTemp": 92,
    "oilPressure": 3.5,
    "fuelLevel": 45,
    "voltage": 13.8,
    "dtcCodes": ["P0301"]
  }
}
```

### Особенности

- ESP32 отправляет данные пачками каждые 1–5 секунд
- При потере связи данные буферизуются на устройстве и отправляются при восстановлении
- Аутентификация: API-ключ устройства в заголовке `X-Device-Key`
- ESP32 не принимает команд от бэкенда (только отправляет)

## 2. Фронтенд (Vue 3) ↔ Бэкенд

**Протокол:** HTTP/HTTPS (REST) + WebSocket
**Формат:** JSON
**Направление:** Двунаправленное
**Инициатор:** Фронтенд (запросы), Бэкенд (push через WebSocket)

### REST API — Фронтенд → Бэкенд

| Эндпоинт                          | Метод | Описание                                    |
|------------------------------------|-------|---------------------------------------------|
| `GET /api/vehicles`                | GET   | Список всех фур с текущим статусом          |
| `GET /api/vehicles/{id}`           | GET   | Детальная информация по фуре                |
| `GET /api/vehicles/{id}/telemetry` | GET   | Телеметрия фуры (с фильтрацией по времени)  |
| `GET /api/vehicles/{id}/alerts`    | GET   | Алерты по фуре                              |
| `GET /api/vehicles/{id}/refuels`   | GET   | Заправки по фуре                            |
| `GET /api/alerts`                  | GET   | Все алерты по всему автопарку               |
| `GET /api/dashboard`               | GET   | Сводка для дашборда                         |

### WebSocket — Бэкенд → Фронтенд

**Подключение:** `wss://server/ws/dashboard`

Бэкенд пушит обновления в реальном времени:

| Событие              | Описание                                      |
|----------------------|-----------------------------------------------|
| `telemetry.update`   | Новые данные телеметрии от устройства          |
| `alert.new`          | Новый алерт (результат AI-анализа)            |
| `device.status`      | Устройство перешло в онлайн/оффлайн           |
| `refuel.new`         | Водитель внёс новую заправку                  |

### Аутентификация

- JWT-токен (получается через `POST /api/auth/login`)
- Роль `admin` — доступ ко всем эндпоинтам на чтение
- Токен передаётся в заголовке `Authorization: Bearer <token>`

## 3. Бэкенд ↔ Telegram Bot API

**Протокол:** HTTPS
**Формат:** JSON (Telegram Bot API)
**Направление:** Двунаправленное
**Инициатор:** Telegram (webhook) и Бэкенд (отправка сообщений)

### Бэкенд → Telegram (отправка алертов водителю)

Бэкенд вызывает Telegram Bot API для отправки сообщений:

| Действие                  | Telegram API метод | Когда                                     |
|---------------------------|--------------------|-------------------------------------------|
| Отправка алерта           | `sendMessage`      | AI обнаружил проблему на CAN-шине         |
| Подтверждение заправки    | `sendMessage`      | После успешного сохранения заправки       |

### Telegram → Бэкенд (команды водителя через webhook)

**Webhook URL:** `POST /api/telegram/webhook`

| Команда водителя        | Что происходит на бэкенде                         |
|-------------------------|---------------------------------------------------|
| `/start`                | Регистрация / привязка Telegram ID к водителю     |
| `/refuel <литры> <руб>` | Создание записи о заправке в PostgreSQL           |
| `/status`               | Бэкенд возвращает текущие параметры фуры водителя |
| `/alerts`               | Бэкенд возвращает последние алерты по фуре        |

### Аутентификация

- Бот авторизуется в Telegram через Bot Token
- Водитель идентифицируется по Telegram ID, привязанному к записи User в БД
- Водитель видит данные только своей фуры (фильтрация по привязке User → Vehicle)

## 4. Бэкенд → AI API (Нейросеть)

**Протокол:** HTTPS
**Формат:** JSON
**Направление:** Бэкенд → AI API (запрос-ответ)
**Инициатор:** Бэкенд

### Когда вызывается

- При получении новой порции телеметрии от ESP32
- Не на каждый пакет — по расписанию или при накоплении батча данных

### Запрос к AI API

```json
{
  "vehicleId": "truck-42",
  "telemetryBatch": [
    {
      "timestamp": "2026-03-22T14:30:00Z",
      "rpm": 1800,
      "coolantTemp": 105,
      "oilPressure": 1.2,
      "dtcCodes": ["P0217"]
    }
  ],
  "context": "Грузовой автомобиль, дизельный двигатель"
}
```

### Ответ от AI API

```json
{
  "anomalies": [
    {
      "severity": "critical",
      "parameter": "coolantTemp",
      "description": "Температура охлаждающей жидкости превышает допустимый порог (105°C)",
      "recommendation": "Остановить движение, проверить уровень антифриза и работу вентилятора"
    }
  ]
}
```

### Что бэкенд делает с ответом

1. Сохраняет алерты в PostgreSQL (таблица `alerts`)
2. Отправляет критические/предупреждения водителю в Telegram
3. Пушит `alert.new` через WebSocket на фронтенд

## 5. Бэкенд ↔ PostgreSQL

**Протокол:** TCP (PostgreSQL wire protocol)
**Направление:** Двунаправленное
**Инициатор:** Бэкенд

### Операции по компонентам

| Источник данных | Операция                | Таблицы                     |
|-----------------|-------------------------|-----------------------------|
| ESP32           | INSERT телеметрии       | `telemetry`                 |
| ESP32           | UPDATE статуса          | `devices`                   |
| Telegram-бот    | INSERT заправки         | `refuels`                   |
| AI API          | INSERT алертов          | `alerts`                    |
| Фронтенд       | SELECT всех данных      | все таблицы (только чтение) |
| Авторизация     | SELECT/INSERT           | `users`                     |
